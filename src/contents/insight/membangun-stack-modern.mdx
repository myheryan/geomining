Membangun website di era sekarang bukan lagi sekadar membuat halaman yang bisa diakses. Ini tentang **pengalaman**, **kecepatan**, dan **estetika**. Dalam artikel ini, kita akan membahas mengapa kombinasi teknologi yang kita gunakan saat ini adalah pilihan terbaik.

## Mengapa Memilih Next.js?

Next.js telah berevolusi menjadi framework yang sangat matang. Berikut adalah beberapa alasan mengapa saya menggunakannya untuk proyek ini:

- **Server Component (RSC):** Memindahkan beban rendering ke server untuk kecepatan akses instan.
- **Optimasi Gambar:** Penanganan otomatis untuk format WebP dan lazy loading.
- **File-based Routing:** Struktur folder yang rapi dan mudah dikelola.
- **SEO Ready:** Meta tags dan struktur HTML yang sangat ramah terhadap mesin pencari.

## Estetika dengan Tailwind CSS

Tailwind CSS memungkinkan kita untuk membangun interface yang sangat kustom tanpa harus menulis file CSS ribet secara terpisah.

> "Utility-first CSS bukan tentang menulis sedikit kode, tapi tentang konsistensi sistem desain."

### Fitur Utama Desain Kita:

1. **Glassmorphism:** Menggunakan `backdrop-blur` untuk memberikan efek kaca transparan yang modern.
2. **Dark Mode:** Transisi warna yang halus antara tema terang dan gelap.
3. **Responsive Grid:** Layout yang otomatis menyesuaikan diri dari layar HP hingga Desktop.

![Visualisasi kode Tailwind CSS yang bersih dengan palet warna Sky Blue dan Slate](https://images.unsplash.com/photo-1587620962725-abab7fe55159?q=80&w=2000&auto=format&fit=crop)

<Callout type="info">
  **Tips:** Gunakan margin negatif pada gambar di MDX untuk menciptakan efek 'breakout' yang membuat pembaca tidak bosan dengan layout yang lurus.
</Callout>

## Alur Kerja Pengembangan

Untuk mencapai hasil yang smooth, saya mengikuti langkah-langkah berikut:

1. Perancangan UI/UX menggunakan Figma.
2. Setup proyek Next.js dengan TypeScript mode ketat.
3. Integrasi MDX untuk pengelolaan konten yang fleksibel.
4. Penambahan micro-interactions menggunakan Framer Motion.
